## 🔍 Технические решения и детали

### Принятое решение: Использовать CLAUDE.md вместо множества файлов

**Контекст:**  
Некоторые проекты используют отдельные файлы для каждого аспекта (ARCHITECTURE.md, TECH_STACK.md и т.д.)

**Варианты:**
1. Один файл CLAUDE.md со всем контекстом
2. Множество файлов (ARCHITECTURE.md, STACK.md, CONVENTIONS.md)

**Выбрали:** Один файл CLAUDE.md

**Причина:**
- Проще для AI - вся информация в одном месте
- Меньше файлов для обновления
- Быстрее загрузка контекста (один `cat` вместо множества)

---

### Принятое решение: PROGRESS.md отдельно от ROADMAP.md

**Контекст:**  
ROADMAP.md уже содержит список задач

**Варианты:**
1. Трекать прогресс прямо в ROADMAP.md (галочки)
2. Отдельный файл PROGRESS.md

**Выбрали:** Отдельный PROGRESS.md

**Причина:**
- ROADMAP.md = план (не меняется)
- PROGRESS.md = статус (меняется постоянно)
- Легче видеть прогресс визуально (progress bars)
- Можно добавлять метрики (velocity, burndown)
