## 🔍 Технические решения и детали

### Проблема 1: PowerShell не видит Git

**Описание проблемы:**  
После установки Git команда `git` не распознавалась в PowerShell

**Решение:**  
Переключились на **Git Bash** вместо PowerShell:
- В терминале VS Code нажали стрелку ▼
- Выбрали "Git Bash"
- Git заработал

**Источник решения:** Claude Chat

---

### Проблема 2: Авторизация GitHub открывалась в неправильном браузере

**Описание проблемы:**  
"Sign in with your browser" открывал Яндекс Браузер, а пользователь залогинен в Chrome

**Решение:**  
Использовали **Personal Access Token** вместо браузерной авторизации:
1. Создали токен на GitHub (scope: `repo`)
2. Использовали токен в URL: `git remote set-url origin https://TOKEN@github.com/...`
3. Push прошёл успешно

**Источник решения:** Claude Chat

---

### Проблема 3: Вставка команд в терминал не работала (Ctrl+V)

**Описание проблемы:**  
Привычный Ctrl+V не работает в Git Bash

**Решение:**  
Используем **правую кнопку мыши** для вставки:
- Скопировал команду (Ctrl+C)
- Кликнул правой кнопкой в терминале
- Команда вставилась

**Альтернативы:** Shift+Insert, Ctrl+Shift+V

**Источник решения:** Claude Chat

## 📚 Что узнал / чему научился

- **Git Bash vs PowerShell:** Git лучше работает в Git Bash
- **Personal Access Token:** Современный способ авторизации GitHub (пароли не принимаются)
- **VS Code терминал:** Удобнее чем отдельное окно командной строки
- **Git workflow:** init → add → commit → remote → push
- **Scope токена:** Нужен именно `repo` для push в репозиторий
- **Разница Claude Chat vs Claude Code:** Чат для планирования, Code для написания кода (установим позже)
