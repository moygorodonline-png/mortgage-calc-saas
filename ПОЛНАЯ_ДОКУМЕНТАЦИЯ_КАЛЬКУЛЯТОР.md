## 📋 ОБЗОР СИСТЕМЫ

### Назначение

Ипотечный калькулятор для расчёта ежемесячных платежей по ипотеке с интеграцией в Битрикс24 для автоматической обработки лидов.

### Основные возможности

- ✅ Расчёт ипотеки по 3 программам (Базовая, Семейная, IT-ипотека)
- ✅ Сравнение предложений банков
- ✅ Валидация лимитов кредита по льготным программам (Семейная — 6 млн ₽, IT — 9 млн ₽)
- ✅ Отправка горячих лидов (имя + телефон) в Битрикс24
- ✅ Отправка холодных лидов (только email) в Битрикс24
- ✅ Отправка расчёта на email
- ✅ Round-robin распределение лидов между менеджерами
- ✅ Интеграция с Yandex.Metrika для отслеживания конверсий
- ✅ Rate limiting для защиты от спама
- ✅ CSV импорт/экспорт данных банков
- ❌ PDF экспорт — не реализован (отложен на v2.0)
- ❌ Отправка расчёта в Telegram — не реализована (отложена на v2.0)
- ❌ Маска телефона (IMask) — не подключена

### Технологии

- **Backend:** PHP 7.4+, WordPress 6.8+
- **Frontend:** JavaScript (ES6), Bootstrap 5, Swiper
- **Database:** MySQL 5.7+
- **Integrations:** Bitrix24 REST API, Yandex.Metrika
- **Plugins:** Meta Box (custom fields)

**Валидация:**
- Email: обязательно, формат email

### Основные функции

#### openCalculator(apartmentData)
Открывает модальное окно калькулятора с данными квартиры. Автоматически запускает тихую загрузку банков для программы `base` в фоне.

**Параметры:**
```javascript
{
    id: 235,
    name: "1-комнатная квартира, 40.30 кв. м.",
    complex: "ЖК Симфония, позиция 2.3",
    price: 5000000,
    url: "https://..."
}
```

**Мобильное поведение:** при ширине < 768px секция `.calculator-lead-section` скрыта до первого расчёта. После расчёта появляется через `slideDown()`.

#### loadBanksSilently(program)
Тихая фоновая загрузка банков без показа лоадера. Использует внутренний кеш `calculator.banks[program]` — повторный запрос к API не делается.

```javascript
loadBanksSilently('base');   // вызывается при открытии
loadBanksSilently('family'); // вызывается при переключении вкладки
loadBanksSilently('it');     // вызывается при переключении вкладки
```

#### calculateMortgageForProgram(program)
Универсальная функция расчёта для любой из трёх программ. Заменяет устаревшую `calculateMortgage()`.

**Алгоритм:**
1. Получение параметров из формы соответствующей программы
2. Расчёт суммы кредита: `цена − взнос − маткапитал`
3. **Проверка лимита кредита по программе** (см. раздел ниже)
4. Запрос к кешу банков: `calculator.banks[program]`
5. Фильтрация банков по минимальному взносу
6. Расчёт платежей для каждого банка
7. Сортировка по платежу (от меньшего к большему)
8. Отображение через `displayResultsForProgram(results, program)`

**Обратная совместимость:** `calculateMortgage()` — обёртка, вызывает `calculateMortgageForProgram('base')`.

#### Валидация лимитов кредита

Применяется в `calculateMortgageForProgram()` перед расчётом.

```javascript
const LOAN_LIMITS = {
    base:   Infinity,  // без ограничений
    family: 6000000,   // 6 млн ₽ (для регионов)
    it:     9000000    // 9 млн ₽ (для регионов)
};
```

**Поведение при превышении:**
- Сумма кредита обрезается до лимита программы
- В начале блока результатов выводится `alert-warning` (Bootstrap) с объяснением и рекомендацией увеличить взнос
- Алерт закрывается крестиком (`alert-dismissible`)
- При повторном расчёте в рамках лимита — алерт не показывается

**Формула расчёта ежемесячного платежа:**
```javascript
function calculateMonthlyPayment(loanAmount, annualRate, years) {
    const monthlyRate = annualRate / 100 / 12;
    const months = years * 12;
    const payment = loanAmount *
        (monthlyRate * Math.pow(1 + monthlyRate, months)) /
        (Math.pow(1 + monthlyRate, months) - 1);
    return Math.round(payment);
}
```

#### displayResultsForProgram(results, program)
Рендерит результаты в контейнер нужной программы.

| Программа | Контейнер |
|-----------|----------|
| `base` | `#banksListContainer` |

## 🔧 TROUBLESHOOTING

### Проблема: Не открывается калькулятор

**Симптомы:** При клике на кнопку ничего не происходит

**Решения:**
1. Проверить консоль браузера (F12) на ошибки JavaScript
2. Убедиться что jQuery загружен
3. Проверить конфликты с другими скриптами
4. Очистить кеш браузера (Ctrl+Shift+R)

### Проблема: Не загружаются банки

**Симптомы:** Пустой список банков или ошибка при расчёте

**Решения:**
1. Проверить что банки импортированы в БД
2. Проверить статус банков (`active`)
3. Проверить консоль на ошибки API
4. Очистить transient кеш:
```php
delete_transient('calculator_banks_base');
delete_transient('calculator_banks_family');
delete_transient('calculator_banks_it');
```

### Проблема: Ошибка при отправке заявки

**Симптомы:** "Ошибка сервера" при отправке формы

**Решения:**
1. Проверить логи: `/wp-content/debug.log`
2. Включить WP_DEBUG в `wp-config.php`:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```
3. Проверить структуру БД (наличие поля `lead_type`)
4. Проверить права доступа к БД

### Проблема: Лид не создаётся в Битрикс24

**Симптомы:** Заявка сохранена в БД, но нет в Битрикс24

**Решения:**
1. Проверить webhook URL в `bitrix24.php`
2. Проверить права вебхука (должны быть: crm, tasks, timeline)
3. Проверить логи:
```bash
grep "Bitrix24" /wp-content/debug.log
```
4. Проверить ID менеджеров (должны существовать в Битрикс24)
5. Проверить интернет-соединение сервера

### Проблема: Письма попадают в спам

**Симптомы:** Email приходит в папку "Спам"

**Решения:**
1. Настроить SMTP (плагин WP Mail SMTP)
2. Настроить SPF/DKIM на хостинге
3. Использовать email на домене сайта (не gmail/mail.ru)
4. Проверить содержимое письма на спам-слова

Подробнее: см. раздел [Email система](#email-система)

### Проблема: Превышен лимит заявок

**Симптомы:** "Превышен лимит заявок. Попробуйте позже."

**Причина:** Rate limiting (10 заявок/час или 5 email/час с одного IP)

**Решения:**
1. Подождать час
2. Изменить лимит в коде (`calculator-api.php`):
```php
if ($recent_leads >= 10) { // Изменить на нужное значение
```
3. Для тестирования временно отключить проверку

### Проблема: Неправильный расчёт платежа

**Симптомы:** Платёж не совпадает с банковским калькулятором

**Причина:** Разные формулы расчёта или округление

**Решения:**
1. Проверить формулу в `calculator-script.js`
2. Проверить ставки банков в БД
3. Учесть что калькулятор не учитывает:
   - Страхование
   - Комиссии банка
   - Досрочное погашение
